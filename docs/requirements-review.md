# 需求评审文档

## 1. 评审目标

本项目目标是建设一套可运行、可落地、接近生产标准的电商秒杀系统，用于 GitHub 发布和项目说明。项目不计划真实上线，但需求、架构、代码、测试、部署、风险预案均按企业项目流程沉淀。

评审重点：

- 明确 C 端商城、C 端秒杀、B 端运营后台的业务边界。
- 明确秒杀高并发场景下的库存、订单、支付、MQ、缓存一致性规则。
- 明确异常场景、限流降级、监控告警和上线前验收标准。
- 明确当前项目不做的范围，避免需求无限扩张。

## 3. 项目范围

### 3.1 C 端商城

- 用户注册、登录、获取当前用户。
- 商品列表、商品搜索、商品详情、SKU 选择。
- 商品图片展示，支持主图和图集。
- 购物车增删改查。
- 普通商品下单。
- 订单详情、订单列表、订单状态展示。
- 模拟支付，支付成功后更新订单和库存。
- 个人中心、会员权益、优惠券展示。

### 3.2 C 端秒杀

- 秒杀活动展示。
- 秒杀场次切换。
- 秒杀商品列表。
- 商品库存进度、秒杀价、原价、限购信息。
- 抢购前领取秒杀令牌。
- 抢购提交携带 `activityId`、`sessionId`、`skuId`、`quantity`、`token`、`requestId`。
- Redis Lua 原子预扣库存。
- RocketMQ 异步建单。
- 秒杀结果轮询。
- 抢购成功后进入订单准备支付页面。

### 3.3 B 端运营

- 秒杀活动管理。
- 秒杀场次管理。
- 活动 SKU 管理。
- 秒杀库存预热。
- 运营操作日志。
- 管理员权限控制。

### 3.4 工程与运维

- Spring Cloud Alibaba 微服务拆分。
- Gateway 统一入口。
- Nacos 服务发现。
- MySQL 持久化。
- Redis 缓存、库存、秒杀令牌。
- RocketMQ 异步削峰。
- Sentinel 限流。
- Prometheus/Grafana 指标观测。
- Flyway 数据库迁移。
- Docker Compose 中间件部署。
- 冒烟测试、压测脚本、上线前检查脚本。

## 5. 核心业务流程

### 5.1 普通购物流程

1. 用户进入商品列表。
2. 搜索或选择商品。
3. 进入商品详情。
4. 选择 SKU 和数量。
5. 加入购物车或立即购买。
6. 创建订单。
7. 锁定库存。
8. 进入模拟支付。
9. 支付成功。
10. 订单状态变为已支付。
11. 库存最终扣减。

### 5.2 秒杀购物流程

1. B 端创建活动、场次和活动 SKU。
2. B 端预热秒杀库存到 Redis。
3. C 端进入秒杀会场。
4. 用户选择正在进行的场次。
5. 前端展示秒杀商品、库存进度和抢购按钮。
6. 用户点击抢购。
7. 前端请求秒杀令牌。
8. 前端携带令牌和 requestId 提交秒杀请求。
9. Gateway 和 Sentinel 对入口流量限流。
10. Redis Lua 校验库存、令牌、重复购买、限购。
11. 预扣成功后投递 RocketMQ。
12. 后端异步消费消息并创建订单。
13. 前端轮询秒杀结果。
14. 成功后进入订单准备支付页。
15. 用户完成模拟支付。

## 6. 关键业务规则

### 6.1 商品与订单

- 商品必须处于上架状态才允许展示和下单。
- SKU 必须有效才允许加入购物车和创建订单。
- 普通订单创建需要 `requestId` 幂等控制。
- 同一个 `requestId` 重复提交不能重复创建订单。
- 待支付订单超时后应自动关闭并释放库存。
- 已支付订单不能重复支付、重复扣库存。

### 6.2 秒杀

- 未开始场次不能抢购。
- 已结束场次不能抢购。
- 活动、场次、活动 SKU 均启用时才允许抢购。
- 同一用户同一活动 SKU 不能超过限购数量。
- 秒杀请求必须携带有效令牌。
- 秒杀库存以 Redis Lua 预扣为准，保证原子性。
- MQ 异步建单必须支持幂等消费。
- 建单失败需要释放 Redis 秒杀库存和用户购买标记。
- 抢购成功只代表订单创建成功，仍需在支付倒计时内完成支付。

### 6.3 支付

- 当前支付为模拟支付。
- 支付成功后更新订单状态。
- 支付成功后触发库存最终扣减。
- 支付回调或支付请求重复到达时必须幂等。
- 真实支付接入不在当前范围内。

## 7. 异常与特殊场景

| 场景 | 处理规则 |
| --- | --- |
| 库存不足 | 返回售罄或抢购失败，不创建订单 |
| Redis 库存已扣但 MQ 发送失败 | 记录失败并通过补偿任务处理 |
| MQ 重复投递 | 消费幂等表或业务唯一键拦截 |
| 异步建单失败 | 记录失败状态，释放秒杀库存 |
| 用户重复点击抢购 | requestId、用户购买标记、限购规则共同拦截 |
| 订单支付超时 | 自动关闭订单，释放库存 |
| 支付重复回调 | 支付单状态机和订单状态机保证幂等 |
| 商品缓存击穿 | 空值缓存、TTL 抖动、重建锁 |
| 高峰流量突刺 | Gateway/Sentinel 限流，Redis 承接秒杀库存校验 |
| Redis 故障 | 秒杀链路降级为暂不可抢 |
| RocketMQ 故障 | 请求受理失败或进入补偿，不直接丢消息 |
| MySQL 故障 | 交易链路不可写时降级，保留只读展示能力 |

## 8. 性能指标

本项目用于本地验收和项目发布，性能指标以可压测、可解释为目标，具体数值需按机器容量调整。

| 指标 | 目标 |
| --- | --- |
| 商品列表接口 | 优先读缓存或分页查询，避免全表扫描 |
| 商品详情接口 | 支持 Redis 缓存、空值缓存、TTL 抖动 |
| 秒杀提交接口 | Redis Lua 快速判定，避免直接打 MySQL |
| 秒杀建单 | RocketMQ 异步削峰 |
| 幂等 | 普通下单、秒杀下单、MQ 消费、支付均需要幂等 |
| 限流 | 秒杀提交、登录、下单接口需要限流策略 |
| 监控 | QPS、RT、错误率、Redis、MySQL、MQ 堆积、JVM 指标可观测 |
